Physical Actuation Trust

Every action earns its authority.

Ori does not treat LLM output as a direct control signal. Every autonomous action is classified into one of four tiers. The tier determines the execution path, oversight requirements, and failure behaviour — governed by the Action Tier Framework.

Tier A — Informational

Always autonomous

WhatsApp alerts, SMS, dashboard logs. The agent reasons and acts without approval. When Ori sends a reasoned alert, no human approved it first.

Tier B — Soft Physical

Autonomous by default

Power source switching, valve modulation, thermostat adjustment. Reversible and low-consequence. Configurable to require approval.

Tier C — Hard Physical

Approval required. Always.

Breaker trips, equipment shutdown, high-pressure valve isolation. Ori reasons and proposes. The operator replies YES or NO. No bypass exists in the architecture.

Tier D — Safety-Critical

Deterministic. Pre-LLM. Inviolable.

Fires from the rule engine before any LLM is consulted. Cannot be disabled by skill config, operator instruction, or runtime state. asyncio.shield() ensures cancellation resistance.

Intelligence Elevator

Cheapest adequate reasoning tier, always.

Four reasoning levels evaluated in order. First adequate tier wins. Tier 1 and 2 operate fully offline.

Tier 1
Rule Engine — Deterministic rule evaluation in microseconds. Handles all Tier D safety-critical events. Always available.
Always
Tier 2
Local SLM — Qwen2.5-0.5B via llama-cpp-python. 3–8s inference on Pi 4. Causal memory lookup before LLM call.
Offline
Tier 3
Gateway LLM — Llama 3.2-3B on a LAN gateway or edge server. MQTT request/response. LAN required.
LAN only
Tier 4
Cloud LLM — Claude API or equivalent. Used only when local reasoning confidence is insufficient.
Optional
